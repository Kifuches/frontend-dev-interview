`Interceptors` (перехватчики) - это, как правило, функции, которые вы можете запускать для каждого запроса и имеют широкие возможности влиять на содержание и общий поток запросов и ответов. Вы можете установить несколько перехватчиков, которые образуют цепочку перехватчиков, где каждый перехватчик обрабатывает запрос или ответ, прежде чем переслать его следующему перехватчику в цепочке.

Вы можете использовать перехватчики для реализации различных распространенных шаблонов, таких как:

- Добавление заголовков аутентификации к исходящим запросам к конкретному API.
- Повторная попытка неудачных запросов с экспоненциальным отсатом.
- Кэширование ответов в течение определенного периода времени или до тех пор, пока они не будут аннулированы мутациями.
- Настройка синтаксика ответов.
- Измерение времени отклика сервера и его регистрация.
- Управляя элементами пользовательского интерфейса, такими как загрузочный спиннер, во время выполнения сетевых операций.
- Сбор и пакетные запросы, сделанные в течение определенного периода времени.
- Автоматически неудачные запросы после настраиваемого срока или тайм-аута.
- Регулярный опрос сервера и обновление результатов.

### Определение перехватчика

```ts
export function loggingInterceptor(req: HttpRequest<unknown>, next: HttpHandlerFn): Observable<HttpEvent<unknown>> {
  console.log(req.url);
  return next(req);
}
```
