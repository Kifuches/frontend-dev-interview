# Change Detection

Change Detection в Angular проверяет, изменились ли данные, от которых зависит шаблон, и синхронизирует DOM с состоянием приложения.

## Default

При стратегии `Default` Angular проходит по дереву компонентов после асинхронных событий: кликов, таймеров, HTTP-ответов и других задач, которые отслеживает Zone.js.

## OnPush

`OnPush` ограничивает проверки компонента. Компонент обычно проверяется, когда:

- изменился reference у `@Input`;
- произошло событие внутри компонента;
- сработал `async` pipe;
- код явно вызвал `markForCheck` или `detectChanges`.

## Что проверяют на интервью

Интервьюеру важно увидеть, что ты не воспринимаешь `OnPush` как магическую оптимизацию. Нужно понимать reference equality, иммутабельные обновления, работу `async` pipe и ситуации, где лишние подписки или мутации ломают ожидаемое поведение UI.
