## Scope

Область видимости (scope) — это часть кода, в которой переменные и функции доступны для использования. В JavaScript область видимости определяет, где можно ссылаться на переменные, и играет ключевую роль в управлении памятью и предотвращении конфликтов имён.

## Виды областей видимости

Кратко:

- Глобальная область: доступна во всем коде.
- Функциональная область: доступна только внутри функции.
- Блочнуая область: доступна внутри конкретного блока, определяемого фигурными скобками.

### Глобальная область видимости

**Описание:** Переменные, объявленные в глобальном контексте (вне всех функций и блоков), доступны во всем коде.

**Особенности:**

- Загрязняет глобальное пространство имён, что может приводить к конфликтам.
- Использование глобальных переменных рекомендуется минимизировать.

```ts
var globalVar = 'I am global'; // или let/const, объявленные вне функций

function showGlobal() {
  console.log(globalVar); // доступ к глобальной переменной
}
```

### Функциональная область видимости (Function Scope)

**Описание:** Переменные, объявленные внутри функции, доступны только внутри этой функции.

**Особенности:**

- Функциональная область видимости помогает изолировать переменные и избегать конфликтов между функциями.
- Ключевое слово var имеет функциональную область видимости.

```ts
function example() {
  var localVar = 'I am local';
  console.log(localVar); // доступно внутри функции
}

console.log(localVar); // ReferenceError: localVar is not defined
```

### Блочная область видимости (Block Scope)

**Описание:** Переменные, объявленные с помощью let или const внутри блока (например, внутри фигурных скобок {}), доступны только в этом блоке.

**Особенности:**

- Позволяет ограничить область видимости переменной внутри условных конструкций, циклов и других блоков.
- Помогает избежать ошибок, связанных с повторным объявлением переменных.

```ts
if (true) {
  let blockVar = 'I am block-scoped';
  console.log(blockVar); // доступно внутри блока
}
console.log(blockVar); // ReferenceError: blockVar is not defined
```

## Особенности работы областей видимости

### Лексическая область видимости

[[orange]Лексическая область видимости]: В JavaScript область видимости определяется во время компиляции (лексически), а не во время выполнения. Это означает, что вложенные функции имеют доступ к переменным внешних функций, даже если они вызываются вне их родительской функции.

#### Состав Lexical Environment

Лексическое окружение состоит из двух частей:

- Environment Record -- хранит все локальные переменные, функции, параметры.
- Ссылка на внешнее лексическое окружение (outer) -- указывает на внешнее окружение, в котором функция была создана (не вызвана!).

Каждая функция, блок `{}`, и глобальная область создают своё лексическое окружение при выполнении.

```ts
function outer() {
  let x = 10;
  function inner() {
    console.log(x); // x доступна здесь благодаря лексической области видимости
  }
  return inner;
}

const innerFunc = outer();
innerFunc(); // 10
```

### Замыкания (Closures)

[[orange]Closure (Замыкания)] - это способность функции помнить lexical environment, в котором она была создана. Это позволяет функции использовать переменные, объявленные в её внешнем окружении.

```ts
function createCounter() {
  let count = 0;

  return function increment() {
    count += 1;
    return count;
  };
}

const counter = createCounter();

console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3
```

Почему работает:

- `createCounter()` завершилась;
- но внутренняя функция `increment` все еще ссылается на переменную `count`;
- поэтому окружение не удаляется сборщиком мусора;
- каждый вызов `counter()` работает с тем же `count`.

Где используется:

- инкапсуляция состояния;
- factory functions;
- callbacks;
- memoization;
- handlers в UI;
- async-код.

Частая задача:

```ts
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
```

Вывод:

```txt
3
3
3
```

Почему: `var` имеет function scope, все callbacks замкнулись на одну переменную `i`.

С `let`:

```ts
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
```

Вывод:

```txt
0
1
2
```

Почему: `let` создает новую binding для каждой итерации цикла.

## IIFE (Immediately Invoked Function Expression)

[[orange]IIFE (Immediately Invoked Function Expression)] – это функция, которая выполняется сразу же после того, как была определена.

Записывается так:

```ts
(function () {
  // какие-то действия
})();
```

IIFE состоит из двух частей:

- Функция с [[blue]лексической областью видимости], заключённая в круглые скобки
- Мгновенно выполняющееся функциональное выражение `()`

Функция внутри скобок создаёт внутри себя область видимости, доступ к которой есть только у неё. Всё, что внутри функции, остаётся только внутри.

Примеры использования:

Используя [[orange]IIFE], можно не бояться конфликтов имён переменных.

```ts
(function () {
  let name = 'Малина';
  console.log(name);
})();

(function () {
  let name = 'Апельсин';
  alert(name);
})();

// Никаких конфликтов
```

Важно: Также переменная, которой присвоено значение IIFE, хранит в себе результат выполнения функции, но не саму функцию.

```ts
let fruitName = (function () {
  let name = 'Малина';
  return name;
})();

console.log(fruitName); // Малина
```

### Коротко:

[[orange]Лексическая область видимости] отвечает на вопрос: Где функция будет искать переменные?

Ответ: В том месте кода, где функция объявлена.

[[orange]Замыкание] отвечает на вопрос: Почему функция все еще имеет доступ к переменным внешней функции, хотя внешняя функция уже завершилась?

Ответ: Потому что функция “захватила” свое внешнее окружение.

#### Можно сказать своими словами на собеседовании так:

Лексическая область видимости в JavaScript значит, что доступ к переменным определяется местом, где функция написана в коде. Внутренняя функция видит переменные внешней функции. А замыкание это ситуация, когда функция продолжает иметь доступ к этим внешним переменным даже после того, как внешняя функция уже завершила выполнение.

И прям совсем бытовая версия:
Функция уносит с собой “рюкзак” с переменными из места, где она была создана.

Только на собеседовании лучше сказать не “рюкзак”, а “лексическое окружение”.
